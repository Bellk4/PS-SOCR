# GLM-OCR Server インストール手順

このドキュメントはインストールと起動準備に特化しています。
運用時の設定値や API の使い方は USAGE.md を参照してください。

## 1. 動作環境

- Python 3.10+
- Windows または Linux/macOS
- CUDA 利用時は対応 GPU + ドライバ
- レイアウト検出利用時は paddleocr（任意）

## 2. 配布・移行時のコピー対象

別サーバーへそのまま移行する場合は、以下をコピーしてください。

- app
- run.bat
- run.sh
- INSTALL.md
- models

補足:

- models をコピーすると、モデル初回再取得を避けやすくなります。
- .venv は環境差分回避のため、移行先で再作成を推奨します。

## 3. クイックスタート

### Windows

```bat
run.bat
```

### Linux / macOS

```bash
chmod +x run.sh
./run.sh
```

起動後:

- UI: http://localhost:8000/
- API Docs: http://localhost:8000/docs

## 4. Python で手動インストール

run.bat / run.sh を使わずに手動でセットアップする場合の手順です。

### Windows (PowerShell)

```powershell
cd <このプロジェクトのパス>
python -m venv .venv
.\.venv\Scripts\Activate.ps1

python -m pip install --upgrade pip

# CPU版
python -m pip install --upgrade --index-url https://download.pytorch.org/whl/cpu torch torchvision

# CUDA版の例
# python -m pip install --upgrade --index-url https://download.pytorch.org/whl/cu126 torch torchvision

python -m pip install fastapi uvicorn websockets wsproto python-multipart pillow pypdfium2 accelerate python-dotenv auth0-server-python httpx
python -m pip install git+https://github.com/huggingface/transformers.git

# 任意（レイアウト検出を使う場合）
python -m pip install --upgrade paddlepaddle paddleocr

python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

### Linux / macOS (bash)

```bash
cd <this-project>
python3 -m venv .venv
source .venv/bin/activate

python -m pip install --upgrade pip

# CPU版
python -m pip install --upgrade --index-url https://download.pytorch.org/whl/cpu torch torchvision

# CUDA版の例
# python -m pip install --upgrade --index-url https://download.pytorch.org/whl/cu126 torch torchvision

python -m pip install fastapi uvicorn websockets wsproto python-multipart pillow pypdfium2 accelerate python-dotenv auth0-server-python httpx
python -m pip install git+https://github.com/huggingface/transformers.git

# 任意（レイアウト検出を使う場合）
python -m pip install --upgrade paddlepaddle paddleocr

python -m uvicorn app.main:app --host 0.0.0.0 --port 8000
```

## 5. Docker ビルド

このプロジェクトのイメージは Linux コンテナです。

### Windows で Linux コンテナ対応 Docker がある場合

```powershell
cd O:\source\Python\PS-SOCR-WEB
docker build -t bellk4/ps-socr-web:20260615 --build-arg TORCH_CHANNEL=cpu .
```

### WSL2 を使わない Windows Server の場合

Linux VM 上の Docker Engine で実行してください。

```bash
cd /path/to/PS-SOCR-WEB
docker build -t bellk4/ps-socr-web:20260615 --build-arg TORCH_CHANNEL=cpu .
```

## 6. Docker 実行（Windows Server / WSL2 なし）

前提:

- Hyper-V が利用可能
- Ubuntu 22.04 などの Linux VM を作成済み
- Linux VM から Docker Hub へ接続可能

### Linux VM で Docker Engine インストール（Ubuntu 例）

```bash
sudo apt-get update
sudo apt-get install -y ca-certificates curl gnupg
sudo install -m 0755 -d /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
sudo chmod a+r /etc/apt/keyrings/docker.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo $VERSION_CODENAME) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
sudo usermod -aG docker $USER
```

再ログイン後に確認:

```bash
docker version
docker run --rm hello-world
```

### 配備・起動

```bash
mkdir -p ~/PS-SOCR-WEB/models/hf_cache ~/PS-SOCR-WEB/models/hf_home

docker login
docker pull bellk4/ps-socr-web:20260615

docker rm -f ps-socr-web 2>/dev/null || true
docker run -d --name ps-socr-web \
  -p 8000:8000 \
  --env-file ~/PS-SOCR-WEB/.env \
  -v ~/PS-SOCR-WEB/models/hf_cache:/app/models/hf_cache \
  -v ~/PS-SOCR-WEB/models/hf_home:/app/models/hf_home \
  --restart unless-stopped \
  bellk4/ps-socr-web:20260615
```

動作確認:

```bash
docker ps
docker logs -f ps-socr-web
```

アクセス先:

- UI: http://<Linux VMのIP>:8000/
- API Docs: http://<Linux VMのIP>:8000/docs
